# 可控步行角色 · 理想资产需求清单
*用于「地平线海岸 ULTRA」（Three.js r169 / GLTFLoader + Meshopt 管线）*

把这份清单直接发给做模型的人即可。带 ✅ 的是必须项，⭐ 是加分项。

---

## 1. 文件格式
- ✅ **glTF 2.0 二进制 `.glb` 单文件**（不要 `.gltf + 外部贴图` 的拆分版）
- ✅ 贴图、动画、骨骼**全部内嵌**进这一个 .glb
- ⭐ 用 **Meshopt** 或 **Draco** 压缩几何（本项目已内置 Meshopt 解码器，可直接吃 Meshopt 压缩的 .glb，体积更小）

## 2. 模型 / 几何
- ✅ **真实比例，单位米**，身高约 **1.7–1.8m**
- ✅ **原点（pivot）在双脚中心、贴地面**（y=0 在脚底）——这样贴地放置最省事
- ✅ **正面朝 +Z**，**Y 轴朝上**（Blender 导出时选 Y-up）
- ✅ 导出前**应用所有变换**（scale=1, rotation=0），无父级偏移
- ✅ 三角面预算 **≈ 1.5万–4万**（要兼顾手机端；越低越好）
- ⭐ 网格尽量合并成**1–2 个 mesh**，减少 draw call

## 3. 材质 / 贴图
- ✅ **PBR 金属-粗糙度**（metallic-roughness）工作流
- ✅ 贴图**内嵌**，单边 **≤ 2048px**（1024 更友好），尽量合并到一张 **atlas**
- ✅ 基础色贴图为 **sRGB**，法线/粗糙/金属为线性
- ⭐ 提供 Normal 贴图提升细节，避免高面数

## 4. 骨骼绑定（Rig）
- ✅ **需要绑定（skinned skeleton）**——可控步行角色必须有骨骼，否则不能播放行走动画
- ✅ **单一骨架**，人形结构，骨骼数 **≤ ~60**
- ✅ 蒙皮权重正常（每顶点 ≤4 骨骼）
- ⭐ 标准人形骨架（如 Mixamo / Humanoid 命名）便于我对应

## 5. 动画片段（最关键）
- ✅ **动画烘焙进同一个 .glb**（导出时勾选 Animations）
- ✅ **原地动画（in-place）——不要带根位移（no root motion）**：人物前进由代码驱动，动画只负责迈腿。带了位移会"滑步/乱飘"
- ✅ **循环无缝**，约 24–30fps
- ✅ 片段**命名清晰**，至少包含：

  | 名称 | 用途 | 必须 |
  |---|---|---|
  | `Idle` | 站立待机 | ✅ |
  | `Walk` | 行走 | ✅ |
  | `Run` | 奔跑 | ⭐ 强烈建议 |
  | `TurnLeft` / `TurnRight` | 原地转身 | ⭐ 可选 |
  | `Jump` | 跳跃 | ⭐ 可选 |
  | `EnterCar` / `Wave` | 上车 / 打招呼 | ⭐ 以后扩展用 |

  > 至少给 **Idle + Walk**，我就能做出能走的角色；加上 Run 手感更好。

## 6. 不要包含
- ❌ 摄像机、灯光（场景自带）
- ❌ 根骨骼位移动画（root motion）
- ❌ 多套 LOD 混在一个文件里（如要 LOD 请单独说明）
- ❌ 外部链接贴图

---

## 最省事的获取方式
如果没有现成角色：去 **Mixamo**（Adobe 免费）选一个人物 → 套用 Idle / Walk / Run 动画 → 分别下载 → 在 **Blender** 里合并成多动画的单个 `.glb` 导出（Y-up、应用变换、勾选 Animations + Meshopt）。Mixamo 的动画**自带原地选项**，正好符合上面要求。
